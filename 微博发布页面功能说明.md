# 微博发布页面功能说明

## 一、功能概述

在微博发布页面，实现以下功能：
1. 读取剪切板第一条内容
2. 在内容结尾加上 `#前端收徒#`
3. 粘贴到输入框（`android.widget.ScrollView`下的`android.widget.EditText`，text是"分享新鲜事..."）
4. 点击发送按钮（`android.widget.LinearLayout`下的`android.widget.TextView`，text是"发送"）

## 二、完整流程

```
进入发布页面
  ↓
等待页面加载（1000ms）
  ↓
读取剪切板第一条内容
  ├─ 成功 → 继续
  └─ 失败（为空） → 跳过输入，直接查找发送按钮
  ↓
在内容结尾加上 #前端收徒#
  ↓
查找输入框
  ├─ 方法1: ScrollView -> EditText（text包含"分享新鲜事"）
  └─ 方法2: 直接查找EditText（text或hint包含"分享新鲜事"）
  ↓
粘贴内容到输入框
  ├─ 方式1: 使用paste()方法
  └─ 方式2: 使用longPressGestureAutoPaste()（备选）
  ↓
等待内容粘贴完成（500ms）
  ↓
查找发送按钮
  ├─ 方法1: LinearLayout -> TextView（text是"发送"）
  └─ 方法2: 直接通过文本查找"发送"
  ↓
点击发送按钮
  ↓
流程完成
```

## 三、核心代码实现

### 1. 读取剪切板方法

```kotlin
/**
 * 读取剪切板第一条内容
 */
private fun getClipboardText(): String {
    return try {
        AssistsService.instance?.getSystemService(Context.CLIPBOARD_SERVICE)?.let { service ->
            val clipboardManager = service as ClipboardManager
            if (!clipboardManager.hasPrimaryClip()) {
                return ""
            }
            clipboardManager.primaryClip?.let { clip ->
                if (clip.itemCount <= 0) return ""
                val item = clip.getItemAt(0)
                // 优先返回文本内容
                item.text?.toString() ?: item.uri?.toString() ?: ""
            } ?: ""
        } ?: ""
    } catch (e: Exception) {
        LogUtils.e(e)
        ""
    }
}
```

### 2. 输入框查找和粘贴

```kotlin
// 方法1: 通过ScrollView -> EditText查找
AssistsCore.findByTags("android.widget.ScrollView").forEach { scrollView ->
    scrollView.findByTags("android.widget.EditText").forEach { editText ->
        val text = editText.text?.toString() ?: ""
        if (text.contains("分享新鲜事") || editText.hint?.toString()?.contains("分享新鲜事") == true) {
            editText.performAction(AccessibilityNodeInfo.ACTION_FOCUS)
            delay(300)
            
            // 粘贴内容
            if (editText.paste(finalText)) {
                overlayLogging("内容粘贴成功")
            } else {
                // 尝试长按粘贴
                editText.longPressGestureAutoPaste(finalText, timeoutMillis = 2000)
            }
        }
    }
}

}
```

### 3. 发送按钮查找和点击

```kotlin
// 方法1: 通过LinearLayout -> TextView查找
AssistsCore.findByTags("android.widget.LinearLayout").forEach { linearLayout ->
    linearLayout.findByTags("android.widget.TextView").forEach { textView ->
        val text = textView.text?.toString() ?: ""

AssistsCore.findByText("发送").forEach { textView ->
    textView.findFirstParentClickable()?.click() ?: textView.click()
}
```

## 四、UI结构说明

### 输入框结构
```
android.widget.ScrollView
  └─ android.widget.EditText
      └─ text: "分享新鲜事..." 或 hint: "分享新鲜事..."
```

### 发送按钮结构
```
android.widget.LinearLayout
  └─ android.widget.TextView
      └─ text: "发送"
```

## 五、关键实现点

### 1. 剪切板读取
- 使用 `ClipboardManager` 读取剪切板
- 获取第一条内容（`getItemAt(0)`）
- 优先返回文本内容，如果没有则返回URI

### 2. 内容处理
```kotlin
val finalText = clipboardText.trimEnd() + " #前端收徒#"
```
- 去除末尾空白字符
- 在末尾添加 `#前端收徒#`

### 3. 输入框定位
- 通过层级结构查找：`ScrollView -> EditText`
- 通过文本/hint匹配："分享新鲜事"
- 备选方案：直接查找所有EditText

### 4. 内容粘贴
- 首先聚焦输入框：`performAction(ACTION_FOCUS)`
- 使用 `paste()` 方法粘贴
- 备选方案：使用 `longPressGestureAutoPaste()`（长按粘贴）

### 5. 发送按钮定位
- 通过层级结构查找：`LinearLayout -> TextView`
- 通过文本匹配："发送"
- 备选方案：直接通过文本查找

### 6. 点击处理
- 优先点击可点击的父节点
- 如果父节点不可点击，直接点击TextView

## 六、错误处理

### 剪切板为空
- 记录日志："剪切板为空，跳过输入"
- 继续查找发送按钮（可能只是发送图片）

### 输入框未找到
- 记录日志："未找到输入框"
- 继续查找发送按钮

### 粘贴失败
- 尝试多种粘贴方式
- 记录详细日志

### 发送按钮未找到
- 记录日志："未找到发送按钮"
- 流程结束

## 七、日志输出

流程中会输出以下日志：

1. `"开始读取剪切板内容"` - 开始读取剪切板
2. `"剪切板内容：xxx"` - 显示剪切板内容
3. `"最终内容：xxx #前端收徒#"` - 显示最终要粘贴的内容
4. `"查找输入框"` - 开始查找输入框
5. `"找到输入框，准备粘贴内容"` - 找到输入框
6. `"内容粘贴成功"` - 粘贴成功
7. `"长按粘贴成功"` - 长按粘贴成功
8. `"未找到输入框"` - 未找到输入框
9. `"查找发送按钮"` - 开始查找发送按钮
10. `"找到发送按钮，准备点击"` - 找到发送按钮
11. `"发送按钮点击成功"` - 点击成功
12. `"微博发布流程完成"` - 流程完成

## 八、延迟说明

- **页面加载等待**：1000ms（1秒）
- **输入框聚焦后等待**：300ms
- **粘贴后等待**：500ms
- **长按粘贴超时**：2000ms

## 十、可能的优化

### 1. 动态等待页面加载
可以检查页面特定元素来判断页面是否加载完成：

```kotlin
var pageLoaded = false
var waitCount = 0
while (!pageLoaded && waitCount < 20) {
    delay(200)
    waitCount++
    if (AssistsCore.findByTags("android.widget.EditText").isNotEmpty()) {
        pageLoaded = true
    }
}
```

### 2. 更精确的输入框定位
可以使用资源ID或其他更精确的属性：

```kotlin
AssistsCore.findById("com.sina.weibo:id/editText")
```

### 3. 内容验证
粘贴后可以验证内容是否正确：

```kotlin
delay(500)
val currentText = editText.text?.toString() ?: ""
if (currentText.contains(finalText)) {
    overlayLogging("内容验证成功")
}
```
