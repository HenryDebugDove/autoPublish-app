# 自动发朋友圈 - 核心代码流程图

## 一、点击"拍照分享"图标 → 打开相册选择

```kotlin
// STEP 5: 点击拍照分享按钮
AssistsCore.findByText("拍照分享").forEach {
    it.click()  // 直接点击文本节点
}

// STEP 6: 点击"从相册选择"
AssistsCore.findByText("从相册选择").forEach {
    it.findFirstParentClickable()?.let {  // ⭐ 关键：找到可点击的父节点
        it.click()
    }
}
```

**流程图**:
```
用户界面: [拍照分享图标] 
         ↓
代码: findByText("拍照分享")
         ↓
找到: AccessibilityNodeInfo (文本节点)
         ↓
执行: click()
         ↓
结果: 弹出选择菜单 [拍照] [从相册选择]
         ↓
代码: findByText("从相册选择")
         ↓
找到: AccessibilityNodeInfo (文本节点，可能不可点击)
         ↓
执行: findFirstParentClickable() ⭐
         ↓
找到: 可点击的父容器 (通常是LinearLayout/RelativeLayout)
         ↓
执行: 父容器.click()
         ↓
结果: 打开相册选择界面
```

## 二、选中相册图片的核心实现

### 方式1: 通过RecyclerView遍历（PublishSocial.kt中使用）

```kotlin
// STEP 8: 选择第一张相片
AssistsCore.findByTags("androidx.recyclerview.widget.RecyclerView").forEach {
    for (index in 0 until it.childCount) {
        val child = it.getChild(index)
        
        // ⭐ 关键判断：RelativeLayout子项中没有TextView = 图片项
        if (TextUtils.equals("android.widget.RelativeLayout", child.className)) {
            child.findByTags("android.widget.TextView").firstOrNull() ?: let {
                // 没有TextView，说明是图片，点击CheckBox选中
                child.findByTags("android.widget.CheckBox").forEach { 
                    it.click() 
                }
            }
        }
    }
}
```

**UI结构分析**:
```
RecyclerView
  ├─ RelativeLayout (相册文件夹项) ← 有TextView("最近")
  ├─ RelativeLayout (图片项1)      ← 无TextView ✓ 选中
  ├─ RelativeLayout (图片项2)      ← 无TextView
  └─ RelativeLayout (图片项3)      ← 无TextView
       └─ CheckBox (选中状态)
       └─ ImageView (图片缩略图)
```

**判断逻辑**:
```
子项是 RelativeLayout?
    ↓ 是
包含 TextView?
    ↓ 否
说明是图片项 ⭐
    ↓
查找 CheckBox
    ↓
点击 CheckBox
    ↓
图片被选中 ✓
```

### 方式2: 增强版图片选择（OverlayBasic.kt中使用）

```kotlin
private suspend fun trySelectImage(container: AccessibilityNodeInfo): Boolean {
    // 方式1: 查找并点击CheckBox
    val checkBoxes = container.findByTags("android.widget.CheckBox")
    checkBoxes.forEach { checkbox ->
        if (!checkbox.isChecked) {
            if (checkbox.click()) return true  // ✅ 成功
        }
    }
    
    // 方式2: 点击ImageView的父容器
    val imageViews = container.findByTags("android.widget.ImageView")
    imageViews.forEach { imageView ->
        imageView.findFirstParentClickable()?.let {
            if (it.click()) return true  // ✅ 成功
        }
    }
    
    // 方式3: 直接点击容器本身
    if (container.isClickable && container.click()) {
        return true  // ✅ 成功
    }
    
    // 方式4: 通过坐标手势点击（最后备选）
    return container.nodeGestureClick()  // 手势模拟点击
}
```

**多重备选机制**:
```
尝试方式1: 点击CheckBox
    ├─ 成功 ✅ → 返回true
    └─ 失败 ❌ → 继续方式2
        ↓
尝试方式2: 点击ImageView父容器
    ├─ 成功 ✅ → 返回true
    └─ 失败 ❌ → 继续方式3
        ↓
尝试方式3: 直接点击容器
    ├─ 成功 ✅ → 返回true
    └─ 失败 ❌ → 继续方式4
        ↓
尝试方式4: 手势点击（坐标模拟）
    └─ 最终结果
```

## 三、核心API调用链

### 1. 文本定位 → 父节点点击

```
AssistsCore.findByText("从相册选择")
    ↓
内部调用: rootInActiveWindow.findAccessibilityNodeInfosByText("从相册选择")
    ↓
返回: List<AccessibilityNodeInfo>
    ↓
遍历每个节点
    ↓
调用: node.findFirstParentClickable()
    ↓
递归向上查找父节点
    ├─ 检查: parent.isClickable == true?
    │   ├─ 是 → 返回parent ✅
    │   └─ 否 → 继续向上查找
    └─ 找到 → 调用: parent.click()
        ↓
执行: parent.performAction(ACTION_CLICK)
    ↓
触发: UI点击事件
```

### 2. 组件类型查找 → CheckBox点击

```
AssistsCore.findByTags("androidx.recyclerview.widget.RecyclerView")
    ↓
内部调用: getAllNodes() → 获取所有节点
    ↓
过滤: className == "androidx.recyclerview.widget.RecyclerView"
    ↓
返回: List<AccessibilityNodeInfo> (RecyclerView列表)
    ↓
遍历每个RecyclerView
    ↓
遍历: recyclerView.childCount (所有子项)
    ↓
检查: child.className == "android.widget.RelativeLayout"
    ↓
在RelativeLayout中查找: findByTags("android.widget.CheckBox")
    ↓
找到CheckBox
    ↓
调用: checkbox.click()
    ↓
执行: checkbox.performAction(ACTION_CLICK)
    ↓
图片被选中 ✓
```

## 四、关键代码片段解析

### 代码1: 为什么需要 findFirstParentClickable()?

```kotlin
// ❌ 错误做法：文本节点可能不可点击
AssistsCore.findByText("从相册选择").forEach {
    it.click()  // 可能失败！
}

// ✅ 正确做法：找到可点击的父节点
AssistsCore.findByText("从相册选择").forEach {
    it.findFirstParentClickable()?.let { parent ->
        parent.click()  // 通过父节点点击
    }
}
```

**原因**:
```
UI结构:
LinearLayout (可点击) ← 真正的点击目标
  └─ TextView ("从相册选择") ← 文本节点，不可点击

如果直接点击TextView → 失败 ❌
点击父LinearLayout → 成功 ✅
```

---

### 代码2: 如何识别图片项？

```kotlin
// 相册列表的结构：
// - 文件夹项：RelativeLayout + TextView("最近"、"相册"等)
// - 图片项：RelativeLayout + ImageView + CheckBox (无TextView)

child.findByTags("android.widget.TextView").firstOrNull() ?: let {
    // 如果没有TextView，说明是图片项
    // 执行选中操作
}
```

**判断逻辑**:
```
RelativeLayout子项
    ↓
查找TextView
    ├─ 找到 → 这是文件夹项，跳过
    └─ 未找到 → 这是图片项 ⭐
        ↓
查找CheckBox并点击
```

---

### 代码3: 多重备选策略

```kotlin
private suspend fun trySelectImage(container: AccessibilityNodeInfo): Boolean {
    // 策略1: CheckBox
    if (找到CheckBox && 点击成功) return true
    
    // 策略2: ImageView父容器
    if (找到ImageView && 点击父容器成功) return true
    
    // 策略3: 直接点击容器
    if (容器可点击 && 点击成功) return true
    
    // 策略4: 手势点击（坐标模拟）
    return nodeGestureClick()
}
```

**为什么需要多种策略？**
- 不同版本的微信UI结构可能不同
- CheckBox可能被遮挡或不可见
- 使用多种方式提高成功率

## 六、关键实现要点速查

| 操作 | 关键代码 | 说明 |
|------|---------|------|
| **查找文本** | `AssistsCore.findByText("文本")` | 在当前窗口查找包含文本的节点 |

## 八、常见问题快速排查

1. **找不到"拍照分享"按钮？**
   - 检查是否已经进入朋友圈页面
   - 检查是否需要等待页面加载
   - 尝试使用 `findByTags()` 通过类名查找

2. **点击没有反应？**
   - 使用 `findFirstParentClickable()` 找到可点击的父节点
   - 检查节点是否可见: `node.isVisibleToUser`
   - 尝试手势点击: `node.nodeGestureClick()`

3. **找不到图片项？**
   - 检查RecyclerView是否正确加载
   - 尝试增加等待时间
   - 使用 `getAllNodes()` 查看所有节点结构

4. **CheckBox点击失败？**
   - 尝试点击图片容器本身
   - 使用手势点击图片中心位置
   - 检查是否有遮挡层
